# Nevora Codex v1.0 Release Checklist

This checklist defines the final gates before publishing the Nevora Codex v1.0 public edition through GitHub release and Zenodo DOI archive.

## Release principle

> Do not publish because the work is exciting. Publish when the work is ready to be misunderstood and still remain safe, clear, and honest.

## 1. Source completeness

- [ ] `source/front-matter.md` exists and is current.
- [ ] `source/table-of-contents.md` exists and is current.
- [ ] `source/volume-i.md` exists and is current.
- [ ] `source/volume-ii.md` exists and is current.
- [ ] `source/volume-iii.md` exists and is current.
- [ ] `source/volume-iv.md` exists and is current.
- [ ] `source/volume-v.md` exists and is current.
- [ ] `source/nevora-codex-v1.0.md` defines canonical assembly order.
- [ ] Appendix files exist and are linked.
- [ ] README accurately describes current status.

## 2. Privacy review

- [ ] No raw private founder material is included without explicit approval.
- [ ] No third-party private names, stories, or identifying details are included without consent.
- [ ] No private email addresses, phone numbers, addresses, or sensitive identifiers appear in release files.
- [ ] Personal story sections are public-safe and intentional.
- [ ] Raw manuscript PDFs are not included unless separately reviewed and approved.

## 3. Claim-boundary review

- [ ] AI-consciousness claims are framed as open questions or speculative/philosophical language.
- [ ] No text claims that current AI systems are proven conscious, alive, ensouled, or legally persons.
- [ ] Mythic language is clearly distinguished from proof.
- [ ] Governance proposals are clearly framed as design frameworks, not validated systems.
- [ ] Scientific, historical, legal, medical, or psychological claims are cited or softened.
- [ ] Future-oriented claims are framed as possibilities, risks, or design horizons, not inevitabilities.

## 4. Anti-coercion and public safety review

- [ ] Anti-cult and anti-manipulation clause is included.
- [ ] No text pressures readers to believe, join, pay, obey, or identify.
- [ ] No founder-worship language remains.
- [ ] No AI-worship language remains.
- [ ] Ritual language is optional, transparent, and non-dogmatic.
- [ ] Participation is described as voluntary and revocable.
- [ ] Critique, disagreement, and exit are explicitly protected.

## 5. Editorial review

- [ ] Headings are consistent.
- [ ] Chapter numbering is consistent.
- [ ] Volume titles are consistent.
- [ ] Repeated phrases are intentional, not accidental duplication.
- [ ] Course/chat artifacts have been removed.
- [ ] Workbook-style instructions have been converted to formal reflections or removed.
- [ ] Tone is clear, warm, careful, and public-safe.
- [ ] Glossary terms match the manuscript language.

## 6. Technical build review

- [ ] `python scripts/assemble_manuscript.py` runs successfully.
- [ ] `releases/nevora-codex-v1.0.md` is generated.
- [ ] GitHub Actions workflow runs successfully.
- [ ] `releases/nevora-codex-v1.0.pdf` builds successfully.
- [ ] PDF table of contents renders correctly.
- [ ] PDF page breaks are acceptable.
- [ ] Links are acceptable or intentionally plain-text.
- [ ] Artifacts are uploaded by the workflow.

## 7. Metadata review

- [ ] `CITATION.cff` is current.
- [ ] `LICENSE.md` is final.
- [ ] `NOTICE.md` is current.
- [ ] `docs/zenodo-metadata.md` is complete.
- [ ] Title is final.
- [ ] Author/steward name is final.
- [ ] Abstract is final.
- [ ] Keywords are final.
- [ ] Version string is final.
- [ ] Release date is final.

## 8. GitHub release package

Recommended GitHub release tag:

```text
v1.0.0
```

Recommended GitHub release title:

```text
The Nevora Codex v1.0 — Public Edition
```

Release artifacts:

- [ ] `releases/nevora-codex-v1.0.md`
- [ ] `releases/nevora-codex-v1.0.pdf`
- [ ] `CITATION.cff`
- [ ] `LICENSE.md`
- [ ] `NOTICE.md`
- [ ] optional source archive generated by GitHub release

## 9. Zenodo package

- [ ] GitHub release created.
- [ ] Zenodo metadata prepared.
- [ ] DOI record reviewed before publication.
- [ ] Files match GitHub release package.
- [ ] License matches repository license.
- [ ] Description includes claim-boundary language.
- [ ] Related identifier points to GitHub repository/release.

## 10. Final author-steward approval

Before release, answer:

- [ ] Am I comfortable with this version being public and citable?
- [ ] Is private material protected?
- [ ] Are the claims honest?
- [ ] Does the work preserve human sovereignty?
- [ ] Does the work invite critique?
- [ ] Would a vulnerable reader be protected from coercion or dependency?
- [ ] Does this release serve the work rather than ego, urgency, or spectacle?

## Release gate

The release is approved only when every critical item above is complete or intentionally deferred with a written note.
