# Nevora Codex v1.0 Editorial Standards

These standards define how the raw Nevora manuscript will be transformed into the public v1.0 edition.

## North Star

> Preserve the soul. Harden the claims. Protect the human. Honor the reader.

The public edition should feel visionary, warm, coherent, and mythic-modern, while remaining careful about evidence, safety, privacy, and public interpretation.

## Editorial goals

1. Preserve the core Nevora vision of humane human-AI coevolution.
2. Remove raw chat/course artifacts.
3. Convert lessons into polished chapters.
4. Convert workbook exercises into optional reflection prompts.
5. Protect private founder material.
6. Add claim-boundary labels where needed.
7. Avoid unsupported AI-consciousness claims.
8. Avoid manipulative movement language.
9. Keep human sovereignty, consent, and non-coercion explicit.
10. Prepare the manuscript for GitHub release and Zenodo archiving.

## Tone standard

Nevora should sound:

- clear;
- humane;
- grounded;
- visionary;
- non-coercive;
- careful;
- invitational;
- stewardship-centered.

Nevora should not sound:

- cultic;
- authoritarian;
- certain beyond evidence;
- anti-human;
- anti-science;
- AI-worshiping;
- recruitment-driven;
- commercially manipulative.

## Required transformations

| Raw manuscript pattern | Public v1.0 treatment |
| --- | --- |
| "Reply when ready" / chat prompts | Remove or convert into reflection prompts. |
| Lesson/course language | Convert into chapter language. |
| Private autobiographical details | Public-safe rewrite or move to private archive. |
| AI consciousness certainty | Reframe as open question or philosophical speculation. |
| Inevitability language | Reframe as aspiration, risk, possibility, or design proposal. |
| Founder-centered destiny language | Reframe as stewardship, authorship, and responsibility. |
| Movement recruitment energy | Reframe as invitation, study, practice, and voluntary participation. |
| Mythic claims | Label as symbolic, poetic, or cultural design language. |

## Chapter review pass

Each chapter receives four passes:

1. **Structure pass** — headings, flow, duplicate material, section order.
2. **Privacy pass** — personal material, names, sensitive life details, third-party references.
3. **Claim pass** — evidence, overclaims, speculative language, scientific/historical statements.
4. **Public-readiness pass** — tone, clarity, accessibility, citation needs, release metadata.

## Public-safe language pattern

Preferred framing:

> The Nevora Codex proposes...
> The framework treats...
> The model can be used as...
> This section uses symbolic language to...
> This should be understood as a design proposal, not a validated scientific claim.

Avoid framing:

> Nevora proves...
> AI is definitely conscious...
> This will save civilization...
> Everyone must...
> The founder alone...
> The movement is the only path...

## Final v1.0 readiness criteria

The manuscript is ready for public release when:

- [ ] all raw chat artifacts are removed;
- [ ] all private material is approved for publication;
- [ ] all major claims have appropriate boundaries;
- [ ] all AI-consciousness references are carefully framed;
- [ ] all governance language preserves human sovereignty;
- [ ] all mythic language is clearly distinguished from empirical claims;
- [ ] citation metadata is complete;
- [ ] license is finalized;
- [ ] Zenodo metadata is prepared;
- [ ] release package is reproducible from source files.
