## After GO

If the decision is GO:

1. Publish GitHub release `v1.0.0`.
2. Attach approved artifacts.
3. Confirm public site/archive links.
4. Prepare Zenodo DOI submission.
5. Update DOI/citation fields after DOI assignment.
6. Open v1.1 feedback cycle.

## If NO-GO or PAUSE

If the decision is NO-GO or PAUSE:

1. Do not publish final release.
2. Record reason in the relevant phase issue.
3. Open or update blocker/major issues.
4. Update `docs/review-response-log.md`.
5. Create a new release candidate only after resolution.

## Closing principle

A GO decision should feel calm, not rushed.

The work should be ready to stand in public without needing the author-steward to explain away its risks.
