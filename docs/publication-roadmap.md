# Nevora Codex Publication Roadmap

**Status:** v1.0 public edition in preparation.

This roadmap defines the path from raw manuscript to public GitHub/Zenodo release.

## Phase 0 — Public scaffold

Goal: establish the repository without publishing raw manuscript materials too early.

Current scaffold:

- `README.md`
- `LICENSE.md`
- `NOTICE.md`
- `CITATION.cff`
- `docs/publication-roadmap.md`
- `docs/claim-boundary-matrix.md`
- `docs/ai-collaboration-disclosure.md`
- `docs/kenotic-covenant-appendix.md`
- `source/.gitkeep`
- `releases/.gitkeep`

## Phase 1 — Manuscript inventory

Collect and organize the current source materials:

- Front matter
- Table of contents
- Volume I — Acts I-IV
- Volume II — Acts V-VIII
- Volume III — Acts IX-XII
- Volume IV — Acts XIII-XVI
- Volume V — Acts XVII-XX

Output:

- source inventory
- duplicated-section report
- private/personal material review list
- glossary seed list

## Phase 2 — Public-safe cleanup

Convert the raw manuscript into a public edition.

Tasks:

- remove chat/course artifacts such as "reply when ready" phrasing;
- convert workbook prompts into formal reflection sections;
- trim or public-safe rewrite private founder details;
- normalize headings, terminology, and act/lesson numbering;
- add claim-boundary labels where needed;
- remove or soften overclaims about AI consciousness, certainty, inevitability, or civilizational outcomes.

## Phase 3 — Claim discipline hardening

Apply the Nevora Claim Boundary Matrix to the full manuscript.

Every major claim should be classified as one of:

- philosophical claim;
- design claim;
- symbolic interface;
- empirical hypothesis;
- operational protocol;
- historical/contextual claim;
- non-claim / poetic expression.

## Phase 4 — Public edition assembly

Create the first v1.0 release candidate.

Outputs:

- `source/nevora-codex-v1.0.md`
- `source/volume-i.md`
- `source/volume-ii.md`
- `source/volume-iii.md`
- `source/volume-iv.md`
- `source/volume-v.md`
- `releases/nevora-codex-v1.0.pdf`
- separate volume PDFs

## Phase 5 — Metadata and citation finalization

Finalize:

- title;
- abstract;
- keywords;
- author/steward metadata;
- license;
- AI collaboration disclosure;
- recommended citation;
- Zenodo upload metadata;
- GitHub release notes.

## Phase 6 — GitHub v1.0 release

Create a tagged GitHub release:

```text
v1.0.0
```

Release package:

- master PDF;
- separate volume PDFs;
- source markdown;
- citation metadata;
- license;
- notice;
- claim-boundary matrix;
- Kenotic Covenant appendix.

## Phase 7 — Zenodo DOI release

Upload the v1.0 GitHub release package to Zenodo.

Recommended record type:

- Publication / Book, or
- Publication / Report, depending on the final framing.

Recommended framing:

> A philosophical, ethical, cultural, and systems-design framework for humane human-AI coevolution, relational intelligence, and stewardship-centered hybrid civilization.

## Phase 8 — v1.1 feedback loop

After public release, collect critique and create a v1.1 roadmap.

Possible additions:

- reviewer response appendix;
- glossary expansion;
- diagrams and infographics;
- GitHub Pages website;
- simplified public guide;
- academic-style summary paper;
- implementation templates.
