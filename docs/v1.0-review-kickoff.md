# Nevora Codex v1.0 Review Kickoff

This document starts the structured v1.0 review cycle.

The repository now has the public manuscript scaffold, safety documents, release workflow, interactive site, and reviewer infrastructure. The next gate is not expansion. The next gate is review.

## Current status

```text
PUBLIC-PREP COMPLETE — STRUCTURED REVIEW OPEN
```

Nevora v1.0 is not final, not DOI-published, and not yet approved for public release.

## Review objective

Determine whether the Nevora Codex v1.0 public edition is ready to become public, citable, and durable without avoidable privacy, safety, claim-boundary, editorial, build, or metadata risks.

## Primary review documents

Start here:

```text
docs/reviewer-guide.md
docs/v1.0-review-plan.md
docs/release-checklist.md
docs/claim-boundary-matrix.md
docs/anti-cult-and-anti-manipulation-clause.md
```

## Review issue queue

The following GitHub issues should be used as review gates:

1. Phase 1 — Author-steward review
2. Phase 2 — Claim discipline review
3. Phase 3 — Privacy and safety review
4. Phase 4 — Editorial and readability review
5. Phase 5 — Build and artifact review
6. Phase 6 — Metadata and license review
7. Phase 7 — Release candidate review

Each phase should end with one of these statuses:

```text
PASS
PASS WITH MINOR EDITS
NEEDS REVISION
BLOCKED
```

## Review rules

- Preserve wonder, but do not overclaim.
- Preserve mythic language, but do not present symbol as proof.
- Preserve AI collaboration language, but do not claim current AI systems are proven conscious.
- Preserve warmth, but do not use flattery or pressure.
- Preserve the founder role, but do not create founder-worship language.
- Preserve public usefulness, but do not expose private material.
- Preserve release momentum, but do not rush into permanence.

## Critical blocker examples

A blocker should be opened or marked if any of these are found:

- private founder or third-party material in public files;
- unsupported AI-consciousness claims;
- language that sounds like doctrine, recruitment, or coercive identity capture;
- professional advice boundary failures;
- serious citation or factual-claim risks;
- broken manuscript assembly or PDF build;
- incorrect license, citation, or Zenodo metadata;
- anything that could reasonably harm vulnerable readers if released unchanged.

## Final gate

The v1.0 public release can proceed only after:

- all phase issues are closed or intentionally deferred;
- no blocker issues remain open;
- generated Markdown and PDF artifacts are reviewed;
- final release notes are prepared;
- author-steward gives final approval.

Final approval phrase:

```text
APPROVED FOR NEVORA CODEX v1.0 PUBLIC RELEASE
```

Until that phrase is recorded, Nevora remains in public-edition preparation.
